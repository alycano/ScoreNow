A continuación, estructuré la API en FastAPI, donde superé desafíos de sintaxis y sangría (indentación) para definir exitosamente dos puntos de acceso (endpoints): uno para leer los equipos existentes (GET /teams) y otro para registrar nuevos equipos (POST /add-team). Finalmente, validamos la conexión y el flujo de datos utilizando herramientas profesionales como el comando curl y la interfaz de documentación interactiva (/docs), confirmando que el servidor ahora es capaz de persistir información en la nube. Con esta base, he pasado de tener un entorno local desconectado a poseer un backend profesional preparado para escalar.
